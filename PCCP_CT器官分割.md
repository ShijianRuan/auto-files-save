### （三）性能评估（Performance Evaluation）

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| **修改项** | **启动性能评估的触发条件** | **评估指标与要素** | **统计分析计划** | **性能目标** | **额外测试需求** |
| 修改项1 | 当调优集结果显示再训练模型在新增器官上的分割精度达到预设阈值时，启动正式测试集验证。预计每12至18个月触发一次。 | 使用与训练数据无重叠的独立测试集进行评估，并保证新增器官与原有器官在测试集中的分布相对均衡。核心指标为DICE系数，样本量按主要终点、等效/非劣效界值和统计功效确定。 | 对新增器官验证其平均DICE系数是否达到预设标准；对原有器官进行等效性检验，证明其性能较已验证版本无临床意义下降（非劣效界值须具备临床合理依据，结合解剖复杂性和临床容忍度论证设定）。 | 验收标准为：（1）新增器官的DICE系数均值达到基于临床可接受性、友商表现确定的通过标准；（2）所有原支持器官性能不劣于已验证版本。 | 以数据库测试为主，辅以软件回归验证。 |
| 修改项2 | 当调优集结果显示再训练模型对所有已支持器官的分割DICE系数达到预设稳定水平后，启动正式测试集评估。预计每12至24个月触发一次。 | 使用独立测试集，综合空间重叠度（如DICE系数）与距离误差等维度的指标评估，重点评价模型在已授权范围内的稳定性和一致性。样本量按预设统计分析计划和功效要求确定。 | 将模型性能与已验证版本进行统计比较，并根据性能变化方向选择优效性或等效性检验，结合分层分析验证确认当前声明场景性能不退化。 | 验收标准为所有已支持器官的DICE系数均值不低于原已验证版本设定的通过标准，且不存在预设不可接受的分层性能下降。 | 以数据库测试为主，辅以软件回归验证。 |
| 修改项3 | 在完成模型架构重构并在调优集上确认基本性能达标后，启动正式测试集评估。预计每18至24个月发生一次。 | 评估使用独立测试集，指标包括各器官DICE系数、推理速度、内存占用和输出稳定性。样本量按预设非劣效检验和功效要求确定。 | 采用非劣效性检验，验证重构模型在分割精度上不劣于已验证版本，并结合稳定性指标确认未引入新的实施风险。 | 验收标准包括：（1）所有器官的DICE系数均值相比原版本无临床意义下降；（2）软件推理延迟的变化不得突破预设的临床工作流可接受时限要求；（3）关键资源占用和输出稳定性满足预设软件验收标准。 | 除数据库测试外，开展必要的软件回归和专项验证。 |
